Đặt vấn đề

– Tới giờ Nginx mới chỉ serve file tĩnh (HTML, CSS, JS, hình ảnh).
– Muốn chạy được PHP (WordPress, Laravel…) thì Nginx không tự hiểu PHP, phải “gửi” file .php tới một PHP engine (thường là php-fpm) qua giao thức FastCGI.

Mục tiêu

– Cài một container PHP-FPM.
– Cấu hình Nginx để nhận request .php và chuyển cho PHP-FPM xử lý.
– Trả kết quả ra cho trình duyệt (ví dụ: file info.php hiện phpinfo()).
